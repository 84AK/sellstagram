# 작업 로그 (Work Log) - 2026.03.25 (2차)

## 📋 작업 개요
- **목표:** 로그인 시 에러 화면 플리커링 및 신규 가입 후 사이드바 미노출 문제 해결
- **담당:** 건축가(Architect), 작업자(Worker), 해결사(Solver)
- **요약:** 인증 콜백 로직을 개선하여 중복 세션 교환 에러를 방지하고, 온보딩 완료 즉시 유저 역할을 상태 저장소에 반영하여 UI 일관성을 확보함.

## 🛠️ 수정 및 구현 내용
### 1. 로그인 플리커링 방지 (`AuthCallback`)
- `src/app/auth/callback/page.tsx`: OAuth 인증 코드 교환 전, 이미 세션이 존재하는지 먼저 확인하는 로직을 추가했습니다.
- 인증 코드 교환 중 에러가 발생하더라도 실제 세션이 생성되었을 수 있으므로, 에러 발생 시 세션을 한 번 더 확인(Retry)하여 불필요한 `/login?error=auth` 리다이렉트를 방지했습니다.

### 2. 사이드바 즉시 노출 (`OnboardingWizard`)
- `src/components/onboarding/OnboardingWizard.tsx`: 온보딩이 완료되는 시점(`handleStudentFinish`, `handleTeacherFinish`)에 `updateProfile`을 호출할 때 `role` 정보를 함께 전달하도록 수정했습니다.
- 이를 통해 새로고침 없이도 사이드바의 렌더링 조건(`user.role` 존재 여부)이 즉시 충족되어 메뉴가 나타나게 됩니다.

## 🐛 해결된 문제 및 검증
- **문제 1:** 구글 로그인 시 "로그인에 문제가 생겼습니다"라는 메시지가 아주 잠깐 보였다가 피드로 이동하는 현상.
- **문제 2:** 신규 가입 완료 후 피드 화면에서 왼쪽 사이드바가 보이지 않고, 새로고침해야만 나타나는 현상.
- **해결:** 두 문제 모두 로직 개선을 통해 해결되었으며, 특히 신규 유저의 첫 사용자 경험(UX)이 매끄럽게 개선되었습니다.

## 📝 향후 참고 사항
- Supabase Auth의 `onAuthStateChange`와 Next.js의 페이지 리다이렉트 간의 경쟁 상태(Race Condition)를 항상 유의해야 합니다.
- 전역 상태(Zustand) 업데이트 시 UI 렌더링에 필수적인 필드(`role` 등)가 누락되지 않도록 주의가 필요합니다.
