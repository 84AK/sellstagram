# 작업 로그 (Work Log) - 2026.03.25

## 📋 작업 개요
- **목표:** 피드 게시물의 캡션(내용) 줄바꿈 가독성 개선
- **담당:** 건축가(Architect), 작업자(Worker)
- **요약:** 사용자가 입력한 줄바꿈(`\n`)이 화면에서도 그대로 유지되도록 CSS 속성을 수정하고, 관련 모든 컴포넌트에 일관되게 적용함.

## 🛠️ 수정 및 구현 내용
### 1. 전역적인 줄바꿈 보존 (`whitespace-pre-wrap`)
- 피드 게시물의 캡션을 렌더링하는 `<p>` 태그에 Tailwind CSS 클래스 `whitespace-pre-wrap`을 추가하였습니다.
- 이 속성은 연속된 공백을 유지하고, 줄바꿈 문자를 만나면 줄을 바꾸며, 컨테이너 너비에 맞춰 자동 줄바꿈을 수행합니다.

### 2. 적용 파일 목록
- `src/components/feed/FeedCard.tsx`: 메인 피드 카드의 캡션 렌더링 부분.
- `src/app/page.tsx`: 랜딩 페이지의 데모 포스트 렌더링 부분.
- `src/app/profile/page.tsx`: 프로필 상세 모달 내 캡션 렌더링 부분.
- `src/app/teacher/page.tsx`: 교사 대시보드 모니터링 피드의 캡션 및 설명 렌더링 부분.

## 🐛 해결된 문제 및 검증
- **문제:** 기존에는 줄바꿈을 입력해도 한 줄로 길게 이어져 가독성이 떨어졌음.
- **해결:** `whitespace-pre-wrap` 적용으로 입력한 서식 그대로 출력됨.
- **검증:** 
    - 줄바꿈이 포함된 긴 텍스트에서 "더보기" 기능이 정상적으로 작동함.
    - 테두리를 벗어나지 않고 적절히 줄바꿈(wrapping)되는 것을 확인.

## 📝 향후 참고 사항
- 이후 새로운 캡션 렌더링 컴포넌트를 추가할 경우, `whitespace-pre-wrap` 클래스를 필수로 포함하여 일관성을 유지해야 함.
- 텍스트가 너무 길어질 경우를 대비해 `line-clamp`와 함께 사용할 때 레이아웃 깨짐이 없는지 주의 필요.
