# 작업 로그 (Work Log) - 2026.03.25 (3차)

## 📋 작업 개요
- **목표:** 피드 로딩 스켈레톤(Shimmer) 애니메이션 추가 및 기존 유저 온보딩 반복 노출 문제 해결
- **담당:** 디자이너(UI Polish), 해결사(Solver), 작업자(Worker)
- **요약:** 2026 트렌드에 맞는 세련된 애니메이션 스켈레톤 UI를 도입하고, 데이터베이스 조회 로직의 견고함을 높여 불필요한 온보딩 진입을 차단함.

## 🛠️ 수정 및 구현 내용
### 1. 애니메이션 피드 스켈레톤 (`FeedSkeleton`)
- `src/components/feed/FeedSkeleton.tsx`: 부드러운 Shimmer(빛 번짐) 애니메이션이 적용된 새로운 스켈레톤 컴포넌트를 제작했습니다.
- `src/app/feed/page.tsx`: 기존의 정적인 로딩 UI를 `FeedSkeleton`으로 교체하여 사용자가 데이터 로딩 중임을 확실히 인지할 수 있도록 개선했습니다.

### 2. 온보딩 반복 노출 방지 (`OnboardingGate`)
- `src/components/onboarding/OnboardingGate.tsx`: 프로필 조회 시 `.single()` 대신 `.maybeSingle()`을 사용하여 결과가 없을 때 발생하는 불필요한 에러 처리를 개선했습니다.
- 프로필 조회 중 에러 발생 시 무조건 온보딩으로 넘기지 않고 에러를 로깅하도록 수정하여, 네트워크 불안정이나 일시적인 DB 지연 등으로 인해 기존 유저가 온보딩 화면을 보게 되는 현상을 방지했습니다.
- 로딩 화면에 "프로필 정보를 확인하고 있어요..."라는 안내 문구를 추가하여 사용자 경험을 보완했습니다.

## 🐛 해결된 문제 및 검증
- **문제 1:** 피드 로딩 시 로딩 중인지 알기 어렵고 시각적으로 심심함.
- **문제 2:** 이미 가입된 유저임에도 로그인 시 온보딩(역할 선택) 화면이 다시 나타남.
- **해결:** 애니메이션 효과 추가로 로딩 가시성을 확보했으며, 프로필 체크 로직 예외 처리를 통해 온보딩 반복 이슈를 해결했습니다.

## 📝 향후 참고 사항
- 스켈레톤 UI는 실제 콘텐츠의 레이아웃과 항상 일치하도록 유지관리해야 합니다.
- Supabase의 `maybeSingle()`은 데이터가 없을 때 `data: null, error: null`을 반환하므로, 이를 활용한 신규 유저 판별이 가장 안전합니다.
