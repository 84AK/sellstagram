# 일반 유저(Viewer) 역할 도입 계획

> 작성일: 2026-03-16
> 상태: 📋 계획 완료 (미구현)

---

## 배경 및 목적

현재 셀스타그램은 학생 전용 마케팅 시뮬레이션 앱으로 설계되어 있다.
학생들이 업로드한 게시물을 외부 링크로 공유했을 때, 일반인도 해당 피드를 보고
좋아요·댓글 등 인스타그램 같은 상호작용을 할 수 있도록 **일반 유저(viewer) 역할**을 도입한다.

---

## 사용자 유형 비교

| 항목 | 학생 (student) | 일반 유저 (viewer) |
|------|--------------|-----------------|
| 가입 방법 | 팀코드 필요 | 이메일/소셜 자유 가입 |
| 홈 화면 | 미션·잔액·XP 대시보드 | 피드 (인스타그램 스타일) |
| 네비게이션 | 피드·미션·세션·샵·프로필 | 피드·검색·프로필 |
| 좋아요·댓글·공유 | ✅ | ✅ |
| 게시물 구매 | ✅ 가상화폐 차감 | ❌ "찜하기"로 대체 또는 "학생만 구매 가능" 안내 |
| 게시물 업로드 | ✅ | ❌ |
| 미션 / 세션 / 샵 | ✅ | ❌ 접근 불가 |
| XP / 잔액 / 랭크 | ✅ | ❌ |

> **구매 처리 결정 필요**: 일반 유저에게 "찜하기"로 대체할지, "학생만 구매 가능" 안내로 처리할지 결정 후 구현

---

## 가입 플로우 분기

```
앱 첫 진입
  ├─ 로그인 없이 → 피드 미리보기 (현재 "/" 홈)
  ├─ 일반 가입 → 이름/아바타 선택 → role: "viewer" → 피드 홈
  └─ 학생 가입 → 팀코드 확인 → 마케터 타입 선택 → role: "student" → 시뮬레이션 홈
```

---

## 로그인 후 라우팅

```typescript
if (role === "viewer")  → /feed  // 간소화된 레이아웃
if (role === "student") → /feed  // 기존 시뮬레이션 레이아웃
if (role === "teacher") → /teacher
```

## 구현 작업 목록

### 1. OnboardingWizard 분기 (핵심)
- 온보딩 첫 화면에 **"학생으로 시작"** / **"일반 가입"** 선택 추가
- 학생: 현재 3단계 유지 (마케터 타입 → 팀코드 → 완료)
- 일반: 2단계로 축소 (이름/아바타 → 완료), `role: "viewer"` 저장

### 2. Sidebar 역할별 탭 분기
- `role === "viewer"` 이면 미션 / 세션 / 샵 탭 숨김
- 일반 유저 전용 상단 브랜드 헤더 표시 (인스타그램 스타일)

### 3. post/[id] 구매 처리
- viewer 역할이면 "구매하기" 버튼 → "찜하기" 또는 "학생만 구매 가능" 안내로 대체

### 4. DB (변경 불필요)
- `profiles.role` 컬럼이 이미 text 타입 → `"viewer"` 값 그대로 사용 가능
